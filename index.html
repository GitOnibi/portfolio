Design methode

  • STEP1. Needs

    사용자의 needs 분석하기 위하여 가장 많은 시간을 소요합니다. 사용자의 needs를 분명히 하였을 때 가장 만족스러운 결과물이 나오기 때문입니다.

    TOOLS

  • STEP2. Concept Design

    파악한 needs를 중심으로 패스트프로토타이핑과 그래픽 작업을 통해 기존의 영역을 재정의 하여 차별화되고 만족스러운 디자인을 이끌어 냅니다.

    TOOLS

  • STEP3. Prototyping

    패스트프로토타이핑 된 결과물을 토대로 needs를 다시 한번 확인합니다. 목표를 분석하고, 목표를 달성하기 위한 세부 기능들을 결정합니다.

    TOOLS

  • STEP4. Coding

    UI/UX를 구현하고 세세한 기능들을 정의하고 연동시킵니다. 다른 분야의 기술자들과의 협업이 요구됩니다.

    • TOOLS

    • LANGUAGE

    • BUILDER

    • FRAMEWORK & LIBRARY

  • STEP5. Testing

    사용자 경험을 중심으로 구현된 기능들의 타당성과 편리성을 고려하고 심미성과 안정성, 크로스 브라우징과 기기호환성 등 성능 최적화 과정을 수행합니다.

    Daum Troy, Browsershots, Moncat, validator etc...

  • STEP6. Complete? not yet.

    어떤 제품에도 완성은 있으나 완벽은 없습니다. 완벽은 이루는 것이 아니라 추구하는 것이고, 변화무쌍하며, 가치는 항상 변합니다. 그것을 인정하고 좇는것이 좋은 디자인이고 웹 프로그래밍입니다.
